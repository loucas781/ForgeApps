API reference

How to connect the CRM to your backend

Base URL: Enter your backend origin only, for example https://forgeapps.co.uk. Do not add a trailing slash.

Authentication model: login should set an HttpOnly session cookie; all subsequent requests are sent with credentials: include.

1) Login

POST /api/developer/session/login

{
  "email": "dev@forgeapps.co.uk",
  "password": "your-password"
}

Expected result: 200 with a developer object and a valid session cookie.

2) Send push

POST /api/developer/session/push/send

{
  "userID": "user_123",
  "title": "Payday update",
  "body": "Your payout has been processed."
}

Expected result: 200/202 when queued successfully.

3) Logout

POST /api/developer/session/logout

{}

Troubleshooting

If login/send fails from browser but works in server tools, verify CORS allows your website origin and also allows credentials.

Required CORS headers usually include Access-Control-Allow-Origin (exact origin, not *) and Access-Control-Allow-Credentials: true.
